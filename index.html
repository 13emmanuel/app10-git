    Selector Universal

    El selector universal, selecciona todos los elementos de un documento HTML. Se denota por un “*”


    Selectores tipo Etiqueta

    Selectores tipo etiqueta o de elementos, son los más simples y comunes.


    Selectores de Clase

    Se utiliza para especificar un estilo a un grupo de elementos con la misma clase. Su principal característica, es que se identifica por un punto.


    Selectores de ID

    Se utiliza para especificar un estilo a un elemento mediante su id. Su principal característica es que se identifica por un sharp o gato “#”


    Diferencia entre Clase e ID

    Clase: se utiliza para un grupo de elementos, es decir como el apellido de nuestros elementos

    ID: se utiliza para un único elemento, no se repite ese id, es decir, como un numero de INE o CURP de nuestro elemento


    Selectores Descendientes

    Afectan a elementos que están contenidos dentro de otro(s) elementos. Lo usamos mediante el uso de un espacio entre cada elemento.

    El diseño responsivo es un diseño que responde al tamaño del dispositivo desde el que se está visualizando la web, adaptando las dimensiones del contenido y mostrando los elementos de una forma ordenada y optimizada.

    Consta de una serie de reglas de diseño web que permiten al contenido de un sitio web adaptarse a diferentes resoluciones en diferentes dispositivos


    Ventajas

  • Reducción de costos de producción
  • Eficiencia en el mantenimiento
  • Mejor usabilidad
  • Impacto en el Visitante

    • Diseño responsivo – Ingredientes

    En el diseño responsivo haremos mucho uso de medidas de porcentaje o medidas relativas tales como “em” , “%”, etc… Aun así es válido poder trabajar con pixeles.

    Estructura Basica De Un Documento HTML


    ETIQUETAS

    El código en HTML está formado por un conjunto de etiquetas (tags). El navegador interpreta estas etiquetas al momento de cargar la página.

    Forma de las Etiquetas HTML

    Símbolos

    Estructura básica:

    nombre de la etiqueta

    Tenemos una amplia gama de etiquetas en HTML, cada una tiene su propio significado. El nombre puede ser escrito en mayúsculas o en minúsculas.

    Pares de etiquetas

    La mayoría de las etiquetas vienen en pares, es decir:

  • Etiqueta Apertura = Inicio
  • Etiqueta Cierre = Final (contienen una diagonal /)

    • Etiquetas Individuales

    Tenemos algunas otras etiquetas que no vienen en pares, es decir, solo tiene etiqueta de apertura por ejemplo:

    Elementos HTML

    Las etiquetas junto con su contenido, forman lo que conocemos como un elemento HTML.



    Elementos HTML

    Todos los elementos HTML se pueden dividir en dos tipos básicos:

  • Elementos en bloque

    • Un elemento en bloque es aquel que contiene un bloque significativo de contenido que debería mostrarse en su propia línea. Por ejemplo: párrafos, tablas, listas, etc…

  • Elementos en línea

    • Los elementos en línea usualmente contienen menos texto y se muestra junto a otro texto en la misma línea, cómo cuando se enfatizan palabras en una oración.

    Los elementos pueden anidarse, es decir, podemos tener elementos HTML dentro de otro(s) elementos

    Atributos en las tags HTML

    La etiqueta de apertura de un elemento puede contener atributos que proporcionan más información sobre el elemento

    Atributos estándar en HTML

    class

    Indica la clase o clases a las cuales un elemento pertenece. Cualquier número de elementos pueden pertenecer a una clase. Un solo elemento puede pertenecer a muchas clases

    style

    Especifica las propiedades CSS de un elemento

    id

    Especifica un identificador único para un elemento

    Comentarios

    Los comentarios en HTML son notas que no se muestran en el navegador y que se pueden leer revisando el código fuente

    El elemento head

    El elemento head contiene información sobre el documento HTML, incluyendo palabras clave, enlaces a otros documentos que usa como archivos CSS y JavaScript.

    A excepción del elemento title que se muestra en la barra de título del navegador, ningún elemento dentro de head se muestra en el contenido de la página HTML

    Elementos de head



  • Title


    • Permite agregar un título a un documento HTML, se muestran en la barra de título de los navegadores y se usa como valor por default cuando se marca una página como favorita.

  • Meta

    • Este elemento contiene información sobre tu página. El término meta se refiere a metadatos, datos de los datos. Su uso más común es para describir información sobre la descripción y palabras clave del documento, y el tipo de codificación.



    Conoce los Atrubtos de Meta

  • script

    • Permite integrar lenguajes de scripting en un documento HTML. Atributos type. Describe el tipo MIME del script que se va a incluir. src: La URL del documento que contiene el script



  • Link

    • define las relaciones entre dos documentos. Su uso más frecuente es para referirse a las hojas de estilo que usa el documento.



  • style

    • Todo el contenido de un documento HTML se encuentra en el elemento body. El elemento body va después del elemento head y debe cerrarse antes de la etiqueta de cierre del elemento html.

    Elementos de body

    Como ya dijimos, en el body estará todo nuestro contenido. HTML5 nos provee una gran variedad de etiquetas para poder darle semántica al contenido

    HTML5 no es una reformulación de versiones previas. HTML5 incluye la redefinición de elementos, la inclusión de nuevos elementos e incluye el uso de nuevas tecnologías como video, audio y canvas.



    Nuevos Elementos

    Nuevas Tags

    Etiqueta hgroup

    Varios headers necesitan múltiples títulos, como un blog que tiene un título y un subtítulo explicando el blog. hgroup permite colocar un h1, h2 y h3 “agrupados” sin afectar el SEO (Search Engine Optimization), permitiendo colocar otro h1 en la página web.


    Etiqueta nav

    Se usa para delimitar una sección que contiene elementos de navegación para el sitio o página. Principalmente está diseñado para que colocar los ‘botones’ de navegación del menú principal. Se puede colocar cualquier etiqueta dentro, aunque lo recomendado es usar listas ul y elementos li.

    Etiqueta section


    Es un conjunto de elementos relacionados que tienen algo en común, por lo cual merecen estar agrupados dentro de un mismo contenedor. En un blog, sería la zona donde están todos los posts. En un video de youtube, habría un section para el video, uno para los datos del video, otro para la zona de comentarios.


    Etiqueta article

    Define zonas únicas de contenido independiente, en otras palabras representa un elemento auto-contenido que puede ser usado o distribuido de forma independiente, por ejemplo el post de un blog, o un artículo en una revista, una noticia.


    Etiqueta aside

    Es un elemento con información escasamente relacionada con el contenido principal.


    Etiqueta footer

    Es el pie de página y todo lo que lo compone pie datos de autor, copyright, fecha de publicación, datos de contacto.

    Formularios

    Un formulario permite a un usuario ingresar datos. La información que se ingresa se manda a un servidor en un grupo de datos que consisten del nombre de los elementos y sus valores.

    Para hacer uso de un formulario, utilizaremos las etiquetas: form Requiere de los atributos action y method, los cuales son la URL del form-handler y el método HTTP por el cual se envían los datos, respectivamente.

  • El valor por default de action es la URL del documento actual
  • El valor por default de method es get (el otro valor válido es post)

    • Partes de un Formulario en HTML5

  • fieldset sirve para agrupar varios controles relacionados
  • legend proporciona un título para un elemento fieldset
  • label crea una etiqueta o nombre para un elemento en específico
  • input Muchos de los elementos de un formulario se crean con este elemento

    • Validación de Formularios

    La validación de un formulario, generalmente es tarea de un form-handler, es decir del lado del servidor. HTML5 nos trae nuevas herramientas las cuales permitirán validar formularios desde el lado del cliente, para dar más eficiencia a la validación.


  • Validación por Valor Faltante

    • Este tipo de validación se ocasiona por el nuevo atributo “required” el cual provoca que si un campo está vacío, se ocasione un error, por lo tanto será inválido.


  • Validación por Patrón

    • Este tipo de validación ocurre cuando nosotros definimos un patrón a seguir dentro de nuestro input

    Este patrón indica que debe de haber 16 caracteres los cuales podrán ser dígitos del0 al 9, caso contrario será inválido.

    ¿Que es CSS?

    Cascading Style Sheets (CSS) ‘Hoja de estilo en cascada’ es un lenguaje usado para definir y crear la presentación de un documento estructurado escrito en HTML. CSS separa la estructura de un documento de su presentación. La información de estilo puede ser definida en un documento separado o en el mismo documento HTML.

    Evolución de CSS

    El organismo W3C (World Wide Web Consortium), encargado de crear todos los estándares relacionados con la web, propuso la creación de un lenguaje de hojas de estilos específico para el lenguaje HTML

    2 Propuestas:

  • CHS (Cascading HTML Sheets) por Hakon Wium Lie
  • SSP (Stream-based Style Sheet Proposal) por Bert Bos

    • Hojas de estilo


    Sintaxis de CSS

    CSS tiene una sintaxis sencilla, que usa palabras clave (inglés) para especificar los nombres de las propiedades y la mayoría de los valores. Se especifica el elemento o elementos a aplicar el estilo (se pueden usar etiquetas, id o clases separadas o se pueden apilar) Vienen en pares de nombre/valor: name:value; Cada línea terminan con ;



    Declaraciones

    Una declaración es un par propiedad y valor que definen la apariencia visual de un elemento HTML.

  • Propiedad:

    • Característica que modifica al elemento seleccionado

  • Valor:

    • Establece el nuevo valor para la propiedad a modificar.
